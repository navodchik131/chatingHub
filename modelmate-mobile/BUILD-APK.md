# Как получить APK для Android (боевой сервер)

Приложение в APK уже настроено на **https://model-mate.online** — тот же сервер, что и веб-кабинет.

## Что нужно один раз

1. Аккаунт на [expo.dev](https://expo.dev) (бесплатный).
2. Node.js на компьютере (у вас уже есть, раз проект запускался).

## Шаги (копируйте команды по порядку)

Откройте **PowerShell** или **cmd**:

```powershell
cd b:\work\chating\modelmate-mobile
npm install
npx eas-cli login
```

В браузере войдите в Expo (или создайте аккаунт). В терминале должно написать, что вы залогинены.

Потом соберите APK (10–20 минут, сборка идёт в облаке Expo):

```powershell
npm run build:apk
```

(отдельно ставить `eas` не нужно — скрипт сам вызывает `npx eas-cli`)

(внутри вызывается `npx eas-cli` — отдельно ставить `eas` не нужно)

При первом запуске Expo спросит:

- **Create a new project?** — ответьте **Yes**.
- Разрешения Android — соглашайтесь по умолчанию.

Дождитесь окончания. В конце будет **ссылка** вида:

`https://expo.dev/accounts/.../builds/...`

## Установка на телефон

1. Откройте эту ссылку **на Android-телефоне** (или скачайте APK на ПК и перекиньте файл).
2. Скачайте **.apk**.
3. Если Android ругается — разрешите установку из неизвестных источников для браузера/файлов.
4. Установите приложение **ModelMate**.
5. Войдите тем же **email и паролем**, что на https://model-mate.online

## Если что-то пошло не так

| Проблема | Решение |
|----------|---------|
| `eas: command not found` | Используйте `npx eas-cli build -p android --profile preview` |
| Сборка упала | Откройте ссылку на build в expo.dev — там текст ошибки |
| Не логинится в приложении | Проверьте интернет на телефоне; логин/пароль как на сайте |
| Картинки не грузятся | Убедитесь, что ставили APK из **preview**-сборки (не старый Expo Go) |

## Пересобрать после изменений в коде

```powershell
cd b:\work\chating\modelmate-mobile
npm run build:apk
```

Новую ссылку на APK снова откроете на телефоне.
